# Synthetic SaaS User Behavior Simulator v2.0

## Documentation Index

Welcome to the documentation for the **Synthetic SaaS User Behavior Simulator**.

This documentation describes the business assumptions, simulation logic, architecture, dataset specification, and quality assurance rules used to generate a realistic B2B SaaS product analytics dataset.

Unlike traditional random data generators, this simulator models user behavior using lifecycle states, personas, hidden behavioral variables, and business scenarios. The goal is to create datasets that closely resemble production analytics data while remaining reproducible and fully configurable.

# Design Philosophy

The simulator follows several guiding principles.

## Business First

Business behavior drives user activity.

Events are generated as a consequence of user decisions rather than random sampling.

---

## State Machine

Each user follows a lifecycle.

```
Signup
    ↓
Exploring
    ↓
Learning
    ↓
Adopting
    ↓
Trial
    ↓
Basic
    ↓
Pro
```

Alternative paths such as dormancy and churn are also supported.

---

## Hidden Variables

User behavior is influenced by internal variables that are **never exported**.

Examples:

- Engagement Score
- Adoption Score
- Health Score
- Intent Score
- Price Sensitivity
- Collaboration Score

These variables determine future decisions during the simulation.

---

## Scenario Driven

Every generated dataset contains one or more predefined business scenarios.

Examples:

- Low-quality Ads
- Weak Onboarding
- Poor Trial Conversion
- Mobile UX Issues
- High Churn
- Strong Organic Growth

These scenarios create meaningful opportunities for analytics and business recommendations.

---

## Controlled Dirty Data

The simulator intentionally injects realistic data quality issues after generating a clean dataset.

Examples include:

- Missing values
- Duplicate events
- Inconsistent labels
- Timestamp drift
- Tracking failures

This enables realistic data cleaning exercises.

---

## Validation

Every generated dataset must pass automated validation before export.

Validation includes:

- Funnel consistency
- Retention sanity checks
- Revenue validation
- Duplicate rate
- Missing value rate
- Lifecycle integrity
