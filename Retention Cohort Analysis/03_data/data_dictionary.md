# 1. Purpose

This document describes the schema of every dataset generated by the simulator.

It serves as the primary reference for analysts, engineers, and dashboard developers.

Business definitions for individual events are documented separately in:

- event_dictionary.md

---

# 2. Data Dictionary Philosophy

The simulator follows a contract-first data design.

Every exported column must have:

- Business meaning
- Data type
- Generation source
- Validation rules

No undocumented column should appear in any exported dataset.

## Schema Change Policy

To preserve compatibility across the analytics pipeline, the following rules apply:

- Existing column names should not be renamed without a migration plan.
- Primary keys must remain immutable.
- Data types should remain stable across versions.
- New columns should be backward compatible whenever possible.
- Deprecated columns should be documented before removal.

---

## Versioning

The simulator follows schema versioning.

Major Version

Breaking schema changes.

Minor Version

New columns or datasets.

Patch Version

Documentation updates and bug fixes only.

Example

Version 2.1.0

• New analytical table added

Version 2.1.1

• Documentation correction

Version 3.0.0

• Breaking schema redesign
