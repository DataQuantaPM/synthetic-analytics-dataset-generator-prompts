# 1. Purpose

This document illustrates how individual users move through the simulated SaaS product.

Rather than describing business rules abstractly, these examples demonstrate realistic user journeys generated by the simulator.

# 2. Journey Overview

Every simulated user begins with the same entry point.

```text
Traffic Source
        │
        ▼
Landing Page
        │
        ▼
Signup
        │
        ▼
Product Journey
        │
        ▼
One of many possible outcomes
```

Different user personas experience different journeys depending on behavior, engagement, and enabled scenarios.

---

# 3. Journey Types

The simulator currently supports the following representative user journeys.

| Journey | Description |
|----------|-------------|
| Healthy User | Converts normally and remains active. |
| Early Churn | Leaves shortly after signup. |
| Power User | Heavy engagement with long retention. |
| Reactivated User | Churns, then returns later. |
| Failed Onboarding | Drops during onboarding. |

These are examples only.

Actual simulations generate thousands of unique variations.

## Event Stream Characteristics

Generated event streams follow these principles:

- Events are strictly chronological within each session.
- Each session begins with a `login` event and typically ends with a `logout` event.
- Every event belongs to exactly one user and one session.
- Event timestamps are internally consistent before dirty data injection.
- Additional metadata (event properties) is attached where relevant.
- Dirty data, such as duplicate events or timestamp drift, is injected only after the clean event stream has been validated.

The resulting event log closely resembles production event tracking data generated by modern SaaS analytics platforms.
