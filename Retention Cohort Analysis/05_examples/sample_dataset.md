This document provides representative examples of the datasets generated by the simulator.

The examples are illustrative only. Actual simulations may produce significantly larger datasets depending on the configured simulation profile.

---

# 2. Dataset Overview

The simulator generates a relational dataset representing the complete lifecycle of SaaS users.

```text
Users
   │
   ├──────── Sessions
   │              │
   │              ▼
   │           Events
   │
   ├──────── Subscriptions
   │              │
   │              ▼
   │           Revenue
   │
   ├──────── Retention
   │
   └──────── Churn
```

All datasets share common identifiers and are designed for SQL joins and BI tools.

# 7. Expected Row Count Formula

The simulator estimates the expected size of each generated dataset based on the simulation configuration.

These estimates are intended for planning, validation, and quality assurance rather than enforcing exact row counts.

Actual outputs may vary slightly due to randomized user behavior, branching logic, and enabled business scenarios.

---

## Row Count Formulas

| Dataset | Expected Formula |
|----------|------------------|
| users.csv | `N_users` |
| sessions.csv | `N_users × Avg Sessions per User` |
| events.csv | `Sessions × Avg Events per Session` |
| subscriptions.csv | `Paid Users × Avg Subscription Lifecycle Records` |
| revenue.csv | `Paid Users × Avg Billing Events` |
| retention_table.csv | `Number of Cohorts × Observation Weeks` |
| churn_table.csv | `N_users` |
| validation_report.json | `1` |

## Scaling Behavior

Dataset growth is not linear across all outputs.

```text
Users
   │
   ├──────── Sessions
   │              (~8× Users)
   │
   ├──────── Events
   │              (~12× Sessions)
   │
   ├──────── Subscriptions
   │              (~22% of Users)
   │
   └──────── Revenue
                  (~3× Paid Users)
```

This hierarchical relationship explains why the event log is typically the largest dataset in the simulation.

## QA Validation

After generation, the Validation Engine compares actual row counts against expected estimates.

| Status | Deviation from Expected |
|--------|-------------------------:|
| PASS | Within ±5% |
| WARNING | ±5–10% |
| FAIL | Greater than ±10% |

Large deviations may indicate:

- unexpected branching behavior,
- incorrect probability configuration,
- excessive filtering,
- simulation bugs,
- invalid scenario interactions.
